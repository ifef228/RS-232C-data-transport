# Виртуальные COM-порты: запуск и тест на одном компьютере

Инструкция для случая, когда **нет второго ПК и нуль-модемного кабеля**: на одной машине создаётся **пара виртуальных последовательных портов**, соединённых «как нуль-модем», и запускаются **две копии** приложения.

---

## macOS (рекомендуется: `socat`)

### 1. Установка

```bash
brew install socat
```

### 2. Создание пары портов

Открой **отдельный терминал** и выполни команду. Окно **не закрывай**, пока тестируешь программу — пока работает `socat`, живёт «виртуальный кабель».

```bash
socat -d -d pty,raw,echo=0,link=/tmp/vcomA pty,raw,echo=0,link=/tmp/vcomB
```

- **`/tmp/vcomA`** и **`/tmp/vcomB`** — удобные имена (симлинки на два конца линии).
- В выводе `socat` будут строки с реальными устройствами, например `/dev/ttys012` и `/dev/ttys013`. Если программа не откроет `/tmp/vcomA`, укажи в поле порта путь **`/dev/ttys0xx`** из этого вывода.

Остановка пары: в этом терминале **Ctrl+C**.

### 3. Запуск приложения дважды

Два отдельных процесса — два окна (два терминала или два запуска из IDE).  
**Логи** дублируются в **терминал** (`[GUI]`, `[PHY TX/RX]`, `[канал]`) — удобно смотреть обмен, если в окне программы не видно деталей.

Запуск из консоли:

```bash
cd /путь/к/RS-232C-data-transport
mvn -q package
java -cp "target/classes:target/lib/*" ru.mgtu.bauman.rs232.Main
```

### 4. Настройки в окнах

Параметры UART (скорость, биты данных, стоп-биты, чётность) на **обоих** окнах должны **полностью совпадать** (например: 115200, 8 бит данных, 1 стоп-бит, без чётности).

| | Окно **приёмника** | Окно **передатчика** |
|---|-------------------|----------------------|
| Режим | Приёмник | Передатчик |
| COM-порт | например `/tmp/vcomB` | например `/tmp/vcomA` |
| Локальный адрес | **2** | **1** |
| Удалённый адрес | **1** | **2** |

Один процесс всегда смотрит в **vcomA**, другой — в **vcomB** (или в два разных `/dev/ttys...` из вывода `socat`).

### 5. Порядок нажатий

1. **Приёмник:** указать **каталог** для сохранения → **Подключить порт**. Ожидание передачи включается **само** (отдельная кнопка «Начать приём» не нужна — в логе появится строка про запуск приёма).
2. **Передатчик:** выбрать текстовый файл → **Подключить порт** → **Передать файл**.

Сначала должен быть подключён **приёмник**, иначе передатчик не получит ACK на кадр Link.

Файл должен появиться в каталоге приёмника под **тем же именем**, что и у исходного файла (без пути).

### 6. Типичные проблемы

| Проблема | Что сделать |
|----------|-------------|
| Порт не открывается | Проверить, что `socat` всё ещё запущен; перезапустить `socat` и оба окна программы. |
| Порт занят | Закрыть оба окна приложения, при необходимости завершить `socat` (Ctrl+C) и создать пару заново. |
| Нет обмена | Проверить адреса (1/2 и 2/1) и что порты **разные** (A и B, не оба в один). |
| Таймауты ACK / «Нет подтверждения для seq=1» | Чаще всего: передатчик нажал «Передать» **до** того, как приёмник подключил порт; приёмник должен быть подключён первым. Также проверь скорость и формат кадра на обоих окнах. В логе приёмника есть строки `[канал] RX type=…` — значит кадры доходят. |

## Windows (альтернатива: `com0com`)

1. Установить драйвер **com0com** (официальный выпуск с SourceForge / проект com0com).
2. Создать **пару виртуальных портов**, например `COM5` ↔ `COM6` (в настройках пары указать, что они соединены друг с другом).
3. Запустить приложение **дважды**: в одном окне указать `COM5`, в другом — `COM6`.
4. Адреса и порядок действий — как в таблице для macOS выше.

Параметры UART снова должны совпадать на обоих окнах.
