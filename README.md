# Пересылка текстовых файлов по RS-232 (вариант 38)

Курсовой проект: локальная «безадаптерная» сеть из двух узлов по **RS-232** (нуль-модем), передача **текстовых файлов**, защита данных **циклическим кодом [15,11]**.  
Стек: **Java 17**, **jSerialComm**, **Swing**.

**Тест на одном ПК (виртуальные порты, socat):** см. отдельный файл **[README-VIRTUAL-PORTS.md](README-VIRTUAL-PORTS.md)**.

---

## Быстрый старт: сборка и запуск

Требуется **JDK 17** и **Maven**.

```bash
cd RS-232C-data-transport
mvn -q package
java -cp "target/classes:target/lib/*" ru.mgtu.bauman.rs232.Main
```

Запуск **два раза** — два окна (два процесса): один приёмник, один передатчик.

Из IDE: главный класс `ru.mgtu.bauman.rs232.Main`.

---

## Два реальных ПК + нуль-модемный кабель

На каждом компьютере — одна копия программы, **разные COM-порты**, те же правила адресов (у одного узла локальный 1 и удалённый 2, у другого — наоборот). Параметры порта на обоих концах должны совпадать.

---

## Что делает программа (кратко)

1. **Физический уровень:** открытие COM-порта, настройка скорости и формата кадра UART, чтение/запись байтов.
2. **Канальный уровень:** кадры с разделителем `0xFF`, адреса, тип, `seq`, полезная нагрузка (экранирование `0xFF`/`0xFE` внутри данных), **ACK** с подтверждением `seq`, повтор при таймауте.
3. **Содержимое файла:** кодируется **циклическим [15,11]** перед разбиением на кадры типа информационного (`FT_I`).
4. **Сценарий передачи:** `Link` → `META` (имя + размер файла в байтах) → серия `I` с кусками закодированных данных → `END` → `Uplink`.

Подробнее о задании и структуре уровней — в методичке кафедры.

## Структура пакетов (в репозитории)

| Пакет / область | Назначение |
|-----------------|------------|
| `physical` | обёртка над COM-портом (jSerialComm) |
| `datalink` | кадры, ACK, очереди приёма |
| `codec` | циклический [15,11], упаковка бит/байт |
| `app` | передача файла, Swing (`MainFrame`) |
| `Main` | точка входа |

---

## Что не эмулируется кодом

- **Cisco Packet Tracer** не подходит для проверки этой программы: порты в симуляторе не открываются в macOS как реальные COM для Java. Для отчёта схему из CPT можно использовать только как **иллюстрацию**, испытания — на железе или через **socat** / **com0com** (Windows).
