# Приложение. Руководство пользователя

**Программа:** пересылка текстовых файлов по RS-232 (курсовой проект, вариант 38).  
**Документ:** инструкция по установке, настройке и работе с программой.

---

## 1. Назначение

Программа позволяет передавать **текстовые файлы** с одного компьютера на другой по **последовательному порту** (интерфейс RS-232, соединение нуль-модемом или через виртуальную пару портов на одном ПК). Имя файла задаётся на передающей стороне; папку сохранения — на принимающей.

---

## 2. Требования к компьютеру

- **Java 17** (JDK) или новее, совместимая с Java 17.
- **Maven** — для сборки из исходных кодов (опционально, если используется готовая сборка).
- Для теста на **одном** компьютере: утилита **socat** (macOS/Linux) или **com0com** (Windows) — см. раздел 6.

---

## 3. Установка и запуск

### 3.1. Сборка из исходников

1. Откройте терминал и перейдите в каталог проекта `RS-232C-data-transport`.
2. Выполните:

```bash
mvn -q package
```

3. Запуск:

```bash
java -cp "target/classes:target/lib/*" ru.mgtu.bauman.rs232.Main
```

### 3.2. Запуск из среды разработки

Укажите главный класс: `ru.mgtu.bauman.rs232.Main`, добавьте в classpath зависимость jSerialComm (после `mvn package` — модули из `target/lib/`).

---

## 4. Общий вид окна

В окне приложения доступны:

- **Режим работы:** «Передатчик (источник файла)» или «Приёмник (каталог для файла)».
- **COM-порт:** имя порта (например, `COM3`, `/dev/ttyUSB0`, `/tmp/vcomA`).
- **Скорость** (боды), **биты данных** (7 или 8), **стоп-биты** (1 или 2), **чётность** (NONE, ODD, EVEN и др.).
- **Адреса:** локальный и удалённый узел (числа 1…126). На двух компьютерах они должны быть **согласованы** (см. раздел 5).
- **Передатчик:** поле пути к **файлу** и кнопка выбора файла.
- **Приёмник:** поле **каталога** для сохранения и кнопка выбора папки.
- Кнопки **«Подключить порт»**, **«Отключить»**, действие передачи или **«Справка: режим приёмника»**.
- Внизу — **журнал** сообщений.

В меню **«Порт»** есть пункт **«Обновить список COM»** — подставляются обнаруженные системой порты; для режима по умолчанию также подставляются рекомендуемые имена виртуальных портов (см. ниже).

---

## 5. Рекомендуемые настройки для двух узлов

Параметры UART (скорость, биты данных, стоп-биты, чётность) на **обоих** компьютерах должны **совпадать**.

| | **Приёмник** | **Передатчик** |
|---|--------------|----------------|
| Режим | Приёмник | Передатчик |
| Порт (виртуальный тест) | `/tmp/vcomB` | `/tmp/vcomA` |
| Локальный адрес | **2** | **1** |
| Удалённый адрес | **1** | **2** |

При переключении режима радиокнопкой программа может автоматически подставить эти порт и адреса.

---

## 6. Работа на одном компьютере (без второго ПК и без кабеля)

1. Установите **socat** (например, `brew install socat` на macOS).
2. В отдельном окне терминала запустите (и не закрывайте во время работы):

```bash
socat -d -d pty,raw,echo=0,link=/tmp/vcomA pty,raw,echo=0,link=/tmp/vcomB
```

3. Запустите программу **дважды** (два окна).
4. **Сначала** настройте приёмник и нажмите **«Подключить порт»** — в логе появится сообщение о запуске приёма.
5. Затем настройте передатчик, **«Подключить порт»**, выберите файл и нажмите **«Передать файл»**.

Подробности — в файле `README-VIRTUAL-PORTS.md` в каталоге проекта.

---

## 7. Работа на двух компьютерах

1. Соедините ПК нуль-модемным кабелем (или адаптерами USB–UART) по схеме из методички.
2. На каждом ПК запустите программу.
3. Укажите **разные** COM-порты, соответствующие вашим кабелям, и **согласованные** адреса как в таблице раздела 5.
4. Одинаковые скорость и формат кадра на обоих концах.
5. Сначала подключите **приёмник**, затем **передатчик** и выполните передачу.

---

## 8. Порядок действий при передаче файла

1. **Приёмник:** указать каталог для сохранения (по умолчанию может быть подставлен каталог запуска программы), нажать **«Подключить порт»**.
2. **Передатчик:** указать путь к текстовому файлу, нажать **«Подключить порт»**, затем **«Передать файл»**.
3. Дождаться сообщения **«Готово»** на передатчике и сообщения о сохранении файла на приёмнике.
4. При необходимости передать **ещё один файл** — снова нажать **«Передать файл»** на передатчике (порт можно не отключать).

## 10. Частые проблемы

| Симптом | Что проверить |
|---------|----------------|
| Сообщение об отсутствии ACK / таймаут | Сначала подключён ли **приёмник**; совпадают ли адреса 1/2 и 2/1; одинаковы ли скорость и формат кадра; работает ли `socat` или кабель |
| Порт не открывается | Правильное имя порта; не занят ли порт другой программой; для виртуальных портов — запущен ли `socat` |
| Файл не появляется | Путь каталога на приёмнике; права на запись в папку |
| Второй файл не передаётся | Использовать версию программы, где после сессии приёмник снова ожидает передачу; при старых сборках — отключить и снова подключить порт |
